Form elements — definitions & simple examples

Each example shows label, live control and the HTML snippet. Keep it simple for learning.

Common attrs: name, id, placeholder, required, readonly
<input type="text" id="name" name="name" placeholder="Enter your name" required>
Masks input; use for credentials.
<input type="password" id="password" name="password" placeholder="Password">
Validates basic email format on submit.
<input type="email" id="email" name="email" placeholder="name@example.com" required>
Use pattern for validation if needed.
<input type="tel" id="tel" name="phone" pattern="[0-9]{10}" placeholder="9876543210">
Use min/max/step attributes for numeric constraints.
<input type="number" id="number" name="qty" min="1" max="10" value="1">
Optimized UI for search inputs on some browsers.
<input type="search" id="search" name="q" placeholder="Search...">
Basic URL format validation on submit.
<input type="url" id="url" name="website" placeholder="https://example.com">
Opens color picker in supported browsers.
<input type="color" id="color" name="favcolor" value="#0ea5a4">
Good for sliders (volume, brightness).
<input type="range" id="range" name="volume" min="0" max="100" value="40">
<input type="date">  <input type="time">  <input type="datetime-local">
Use accept to restrict file types; use multiple for multiple files.
<input type="file" name="avatar" accept="image/*">
<input type="checkbox" name="features" value="wifi"> <input type="radio" name="size" value="m">
<textarea id="message" name="message" rows="4" cols="40"></textarea>
Supports single & multiple (multiple attribute).
<select name="country"> <option value="in">India</option> </select>
<input list="browsers"> <datalist id="browsers"> <option value="Chrome"> </datalist>
<button type="submit">Submit</button>

Short notes

  • Use label for="id" or wrap the control in <label> for accessibility.
  • Give each control a name attribute to include it in form submissions.
  • Use required, disabled, readonly where appropriate.
  • Use accept for file types, and min/max/step for numbers and ranges.
  • Prefer semantic markup and provide clear placeholder or helper text.
Keep examples small and practice building a real form (login, profile, product review) to remember attributes and accessibility.
